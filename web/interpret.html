Understanding Your Results

How to interpret FUVA test results and recommendations

Dashboard Overview

The dashboard shows summary statistics from all your tests:

  • Total Runs - Number of test sessions you've run
  • Completed - Tests that finished successfully
  • Tests Run - Total individual file uploads
  • Anomalies - Security issues detected

Understanding Anomalies

Anomalies are potential security issues found during testing. They are categorized by severity:

Severity Levels

CRITICAL Immediate security risk. Dangerous file was accepted.
HIGH Significant security issue that needs attention.
MEDIUM Moderate risk, should be addressed.
LOW Minor issue, may not require immediate action.
INFO Informational, no immediate action needed.

Types of Anomalies

Security Dangerous files accepted, exploit patterns detected
Performance Slow response times, timeouts
Validation Missing or weak file validation
Configuration Server misconfiguration detected

Test File Categories

FUVA tests your upload endpoint with these categories of files:

Images

  • Corrupt PNG/JPEG - Files with broken headers
  • Large Metadata - Images with excessive EXIF data
  • Wrong MIME - Images with incorrect content-type
  • Empty Image - Zero-byte image files
  • Giant Image - Extremely large image dimensions

Documents

  • Malformed PDF - PDFs with broken structure
  • SVG with XSS - SVG files containing scripts
  • PDF with JavaScript - PDFs with embedded scripts
  • RTF exploits - Rich Text Format vulnerabilities
  • Office with macros - Word/Excel with VBA macros

Archives

  • Zip Slip - Archives with path traversal
  • Archive Bombs - Highly compressed files (ZIP, 7z, XZ, BZ2)
  • Nested Archives - Multi-level nested archives
  • Symlink Archives - Archives with symbolic links
  • Web Shell Archives - Archives containing backdoors

Executables

  • Fake EXE/ELF - Executables with wrong extensions
  • Double Extensions - file.txt.exe
  • Null Byte Injection - file.php%00.jpg

Other Tests

  • Polyglot Files - Files that are multiple types
  • XXE - XML External Entity attacks
  • Serialization Attacks - Python/PHP pickle exploits
  • Command Injection - Files with shell commands

Status Codes Explained

Success Codes (Good)

  • 200 OK - Request succeeded
  • 201 Created - File was uploaded and stored

Error Codes (Usually Good)

  • 400 Bad Request - Validation caught the file
  • 403 Forbidden - Access denied
  • 413 Payload Too Large - Size limit enforced
  • 415 Unsupported Media Type - File type blocked

Error Codes (Warning)

  • 200 OK for dangerous files - VULNERABILITY!
  • 500 Server Error - Possible crash/bug

Recommendations

If dangerous files are accepted:

  1. Implement content-based file type detection (not just extension)
  2. Use magic byte/mime type validation
  3. Store files outside web root
  4. Rename uploaded files to random names
  5. Set proper file permissions

If validation is weak:

  1. Implement server-side validation (never trust client)
  2. Check file headers, not just extensions
  3. Limit file size
  4. Use allowlists instead of blocklists

If you see errors:

  1. Check server logs
  2. Review error messages for information disclosure
  3. Implement proper error handling

Comparing Results

Use the comparison feature to:

  • Compare results before/after security fixes
  • Test different upload endpoints
  • Verify configuration changes
Tip: Select two runs from the Recent Runs table and click "Compare Selected" to see the differences.

Export Formats

FUVA provides multiple export formats:

  • HTML Report - Visual report with charts (click "View")
  • JSON Report - Machine-readable data
  • CSV Export - Spreadsheet compatible

CSV export: Add ?format=csv to the report URL.

Security Best Practices

  1. Never trust client-side validation - Always validate on server
  2. Use allowlists - Only allow known-good types
  3. Store outside webroot - Don't serve uploaded files directly
  4. Randomize filenames - Prevent path traversal
  5. Scan for malware - Use antivirus on uploaded files
  6. Limit size - Prevent denial of service
  7. Log and monitor - Track upload activity
